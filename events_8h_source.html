33 // Events are emitted by peers at a variety of points during the simulation.
34 // Each event is emitted by a particlar peer at a particular time. Collectors
35 // process these events, perhaps calculating statistics or storing events to
36 // a log for post-processing.
37 //
38 // The Event types can be arbitrary, but should be copyable and lightweight.
39 //
40 // Example collectors can be found in collectors.h, but have the general
41 // interface:
42 //
43 // @code
44 // template <class T>
45 // struct Collector
46 // {
47 // template <class Event>
48 // void
49 // on(peerID who, SimTime when, Event e);
50 // };
51 // @endcode
52 //
53 // CollectorRef.f defines a type-erased holder for arbitrary Collectors. If
54 // any new events are added, the interface there needs to be updated.
