Construct a STValidation from serialized data previously shared by a peer.

Parameters

Construct, sign and trust a new STValidation.

Constructs, signs and trusts new STValidation issued by this node.

Parameters
ledgerHashThe hash of the validated ledger
ledgerSeqThe sequence number (index) of the ledger
consensusHashThe hash of the consensus transaction set
signTimeWhen the validation is signed
publicKeyThe current signing public key
secretKeyThe current signing secret key
nodeIDID corresponding to node's public master key
isFullWhether the validation is full or partial
feeFeeSettings to include in the validation
amendmentsIf not empty, the amendments to include in this validation
Note
The fee and amendment settings are only set if not boost::none. Typically, the amendments and fees are set for validations of flag ledgers only.
